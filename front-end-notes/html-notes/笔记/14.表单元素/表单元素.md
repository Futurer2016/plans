## input 元素 空元素

输入框

type属性: (输入框类型)
常见取值:
1. text: 普通文本输入框
2. password: 密码框
3. date: 日期选择框(有兼容性问题), 调取系统日期选择框，或者浏览器实现的日期选择框
4. search: 搜索框(有兼容性问题)，手机上的搜索框会导致键盘按键发生变化(有搜索按键)
5. range: 滑块, 配套属性min="0" max="5"
6. color: 颜色选择,调取系统颜色选择器，或者浏览器实现的颜色选择器
7. number: 数字输入框(有一点兼容性问题), 配套属性min="0" max="5" step="1"(步进)
8. checkbox: 多选框 要求必须有name属性，表示一组多选框
9. radio: 单选框 要求必须有name属性，表示一组单选框
10. file: 选择文件 弹出文件选择框
11. button: 普通按钮
12. submit: form表单中，提交按钮
13. reset: form表单中，重置按钮

value属性: (输入框的值)

placeholder: 显示提示的文本，文本框没有内容时显示

## select 元素
下拉列表选择框, 通常有多个option子元素配合使用

option 元素有一个布尔属性 selected 表示默认选中的项目

> 给下拉列表进行分组
> optgroup作为select的子元素表示分组选择，通常有多个option子元素配合使用

>> select 有 布尔属性 multiple 表示可以多选，操作时按住键盘的ctrl  或者  shift 键可以进行多选

## textarea 文本域 / 多行文本框 可替换元素

## 按钮元素 button html4.01版本提出的元素 几乎没有兼容性问题, 语义上表示按钮

type 属性: reset / submit(默认值) / button

## 配合表单元素的其他元素

### label 普通元素
通常配合表单元素

- 显式关联: 可以通过for属性，让label元素关联某一个表单元素，for属性关联表单元素id值
- 隐式关联: label元素里面嵌套表单元素作为子元素

### datalist 比较新的结构，有兼容性问题
数据列表
该元素不会显示在页面中，见示例。

### form 元素 

通常，会将整个表单元素，放置在form元素的内部，作用是当提交表单时，会将form元素内部的表单内容，以合适的方式提交到服务器。

form元素对开发静态页面没有什么意义。

### fieldset 元素

表单分组，见示例

### 表单状态

readonly 布尔，只读，不会改变表单显示样式

disabled 布尔，禁用，会改变表单显示样式
